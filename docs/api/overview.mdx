The Misinfo Research API is a REST API that lets you programmatically manage misinformation studies, configure posts, and collect participant data from your self-hosted instance. All requests are made directly to your host with no base-path prefix — every endpoint lives at the root of your domain.

## Base URL

```
https://your-host
```

Replace `your-host` with the domain or IP address of your self-hosted Misinfo Research instance. All paths in this reference (e.g. `/api/admin/studies`) are appended directly to this base URL.

## API Groups

The API is divided into two groups with different authentication requirements.

**Admin API** — endpoints under `/api/admin/` that let researchers and administrators manage studies, posts, and export data. All admin endpoints require a valid JWT Bearer token in the `Authorization` header. See [Authentication](/api/authentication) for details.

**Participant API** — endpoints under `/api/session/`, `/api/reaction`, and `/api/rating` that power the participant-facing experiment flow. These endpoints require **no authentication**. They are designed to be called directly from your front-end or survey integration.

## Content Types

| Scenario | Request `Content-Type` |
|---|---|
| All admin and participant endpoints (except image upload) | `application/json` |
| Image upload (`POST /api/admin/posts/:id/image`) | `multipart/form-data` |

All responses are returned as `application/json` unless the endpoint returns a file download (export endpoints return `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` or `text/csv`).

## Error Format

Every error response uses a consistent JSON envelope:

```json
{ "error": "Human-readable description of what went wrong" }
```

### HTTP Status Codes

| Code | Meaning |
|---|---|
| `200` | Success |
| `400` | Bad request — missing or invalid fields in the request body |
| `401` | Unauthorized — missing, invalid, or expired token or session token |
| `403` | Forbidden — authenticated but insufficient role (e.g. researcher accessing admin-only operation) |
| `404` | Not found — study inactive, wrong slug, or cross-user access attempt |
| `409` | Conflict — duplicate resource (e.g. duplicate username) |

<Note>
A `404` can also be returned for security reasons when a study exists but the authenticated user does not have permission to access it. This prevents enumeration of other users' studies.
</Note>

## Endpoint Reference

The table below lists every endpoint in the API with its authentication requirement and a short description.

| Method | Path | Auth | Description |
|---|---|---|---|
| `POST` | `/api/admin/login` | None | Exchange credentials for a JWT token |
| `GET` | `/api/admin/studies` | Bearer JWT | List studies visible to the authenticated user |
| `POST` | `/api/admin/studies` | Bearer JWT | Create a new study |
| `PATCH` | `/api/admin/studies/:id` | Bearer JWT | Update study settings |
| `DELETE` | `/api/admin/studies/:id` | Bearer JWT | Delete a study and all its data |
| `POST` | `/api/admin/studies/:id/duplicate` | Bearer JWT | Copy a study with all posts but no sessions |
| `GET` | `/api/admin/studies/:id/posts` | Bearer JWT | List posts belonging to a study |
| `POST` | `/api/admin/posts` | Bearer JWT | Create a new post in a study |
| `PATCH` | `/api/admin/posts/:id` | Bearer JWT | Update post fields |
| `PATCH` | `/api/admin/posts/:id/reorder` | Bearer JWT | Move a post up or down in its study |
| `POST` | `/api/admin/posts/:id/image` | Bearer JWT | Upload a post image (multipart) |
| `DELETE` | `/api/admin/posts/:id/image` | Bearer JWT | Remove the image from a post |
| `GET` | `/api/admin/dashboard/:studyId` | Bearer JWT | Retrieve aggregated dashboard statistics |
| `GET` | `/api/admin/export/:studyId` | Bearer JWT | Download session data as XLSX or CSV |
| `POST` | `/api/session/start` | None | Start a participant session and receive the study bootstrap |
| `POST` | `/api/session/consent` | None | Record participant consent decision |
| `POST` | `/api/session/demographics` | None | Submit participant demographic responses |
| `POST` | `/api/reaction` | None | Record a participant reaction to a post |
| `POST` | `/api/rating` | None | Record a credibility rating for a post (classic studies) |
| `POST` | `/api/session/complete` | None | Mark a session complete and receive debrief content |
