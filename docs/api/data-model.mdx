This page describes the shape of the key data objects used throughout the Misinfo Research API. Understanding these structures helps you parse responses correctly, build reliable front-end rendering logic, and map collected data to your analysis pipeline.

---

## Session

The Session object is returned in full by `POST /api/session/start`. It is the single bootstrap payload your front-end needs to render the entire experiment without any further API calls.

<ResponseField name="session_token" type="string">
  UUID that uniquely identifies this participant's session. Include this value in every subsequent participant API call (`/api/session/consent`, `/api/reaction`, etc.).
</ResponseField>

<ResponseField name="is_preview" type="boolean">
  `true` if this session was started with `preview: true`. Preview sessions are excluded from analysis exports.
</ResponseField>

<ResponseField name="style_condition" type="string">
  The visual style variant assigned to this participant (e.g. `"A"` or `"B"`). Determined at session creation before consent.
</ResponseField>

<ResponseField name="metric_condition" type="string">
  The engagement-metric display variant assigned to this participant (e.g. `"shown"` or `"hidden"`).
</ResponseField>

<ResponseField name="full_condition" type="string">
  Compound condition string combining style and metric conditions (e.g. `"A_hidden"`). Useful as a single grouping key in analysis.
</ResponseField>

<ResponseField name="external_id" type="string">
  Identifier derived from `url_params` (e.g. a Prolific or MTurk worker ID). Use this to join session records back to your participant panel.
</ResponseField>

<ResponseField name="language" type="string">
  BCP 47 language code of the study (e.g. `"en"`, `"de"`, `"fr"`).
</ResponseField>

<ResponseField name="locale" type="object">
  <Expandable title="locale fields">
    Key-value map of all participant-facing UI strings in the study's language. Keys are semantic identifiers (e.g. `"like"`, `"share"`, `"next_post"`) and values are the translated strings to render in the interface.
  </Expandable>
</ResponseField>

<ResponseField name="parts" type="array">
  <Expandable title="part object fields">
    Array of study part objects for multi-part experiments. Each part object includes:

    <ResponseField name="id" type="number">
      Integer ID of the part.
    </ResponseField>

    <ResponseField name="name" type="string">
      Display name for this part of the study.
    </ResponseField>

    <ResponseField name="instruction_text" type="string">
      Instructions shown to the participant at the start of this part.
    </ResponseField>

    <ResponseField name="part_order" type="number">
      Position of this part within the study (1-based).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="posts" type="array">
  Ordered array of Post objects (see [Post](#post) below) to present to this participant.
</ResponseField>

<ResponseField name="demographic_questions" type="array">
  <Expandable title="demographic question fields">
    Array of demographic question configuration objects defining which questions to collect and in what format. Each object includes a `key`, `type`, `label`, and optional `options` array for multiple-choice questions.
  </Expandable>
</ResponseField>

<ResponseField name="post_questions" type="array">
  <Expandable title="post question fields">
    Array of per-post question configuration objects shown after each post (or at the end of the study, depending on `demographics_position`). Each object includes a `key`, `type`, `label`, and display order.
  </Expandable>
</ResponseField>

<ResponseField name="study" type="object">
  Study display settings object. See [Study Display Object](#study-display-object) below.
</ResponseField>

## Post

Post objects appear in the `posts` array of the Session bootstrap. Each object describes a single feed item to present to the participant.

<ResponseField name="id" type="number">
  Integer ID of the post. Include this when submitting reactions and ratings.
</ResponseField>

<ResponseField name="post_order" type="number">
  1-based position of this post in the participant's feed. Include this when submitting reactions and ratings.
</ResponseField>

<ResponseField name="part_id" type="number">
  ID of the study part this post belongs to. Use this to group posts by part in multi-part studies.
</ResponseField>

<ResponseField name="part_ids" type="array">
  Array of part IDs this post is associated with, for posts that span multiple parts.
</ResponseField>

<ResponseField name="topic" type="string">
  Topic label for the post (e.g. `"climate"`). Hidden from the participant when `hide_topic` was set to `true` at post creation.
</ResponseField>

<ResponseField name="emoji" type="string">
  Emoji displayed alongside the topic label (e.g. `"🌍"`).
</ResponseField>

<ResponseField name="headline" type="string">
  The resolved headline for this participant — either `headline_a` or `headline_b` depending on the assigned variant.
</ResponseField>

<ResponseField name="content" type="string">
  The resolved body text for this participant — either `content_a` or `content_b` depending on the assigned variant.
</ResponseField>

<ResponseField name="source_name" type="string">
  Display name of the simulated post source (e.g. `"The Daily Tribune"`).
</ResponseField>

<ResponseField name="source_handle" type="string">
  Social-media handle of the simulated source (e.g. `"@dailytribune"`).
</ResponseField>

<ResponseField name="time_ago" type="string">
  Relative timestamp string to display on the post (e.g. `"3 hours ago"`).
</ResponseField>

<ResponseField name="is_true" type="boolean">
  Ground-truth label for the post (`true` = accurate content, `false` = misinformation). **Never display this value to participants.**
</ResponseField>

<ResponseField name="image_url" type="string | null">
  URL of the post's attached image, or `null` if no image has been uploaded.
</ResponseField>

<ResponseField name="avatar_url" type="string | null">
  URL of the source's avatar image, or `null` if none is set.
</ResponseField>

<ResponseField name="likes_shown" type="number">
  Initial like count to display to this participant (base count adjusted for any cumulative session effects).
</ResponseField>

<ResponseField name="shares_shown" type="number">
  Initial share count to display.
</ResponseField>

<ResponseField name="dislikes_shown" type="number">
  Initial dislike count to display.
</ResponseField>

<ResponseField name="flags_shown" type="number">
  Initial flag count to display.
</ResponseField>

<ResponseField name="manipulation_techniques" type="array">
  <Expandable title="manipulation technique fields">
    Array of objects describing the persuasion or manipulation techniques present in this post. Each entry includes a `key` and `label`. Used for educational debrief overlays when enabled.
  </Expandable>
</ResponseField>

<ResponseField name="show_like" type="boolean">
  Whether the like button should be rendered for this post.
</ResponseField>

<ResponseField name="show_dislike" type="boolean">
  Whether the dislike button should be rendered for this post.
</ResponseField>

<ResponseField name="show_share" type="boolean">
  Whether the share button should be rendered for this post.
</ResponseField>

<ResponseField name="show_flag" type="boolean">
  Whether the flag button should be rendered for this post.
</ResponseField>

<ResponseField name="show_comment" type="boolean">
  Whether the comment input should be rendered for this post.
</ResponseField>

<Note>
Always use the per-post `show_*` flags to conditionally render interaction buttons. These may differ between posts within the same study.
</Note>

## Reaction

The Reaction object is what you send to `POST /api/reaction`. It is not returned by any read endpoint but its fields are present in the data export. The table below summarises the `action` enum values and their meanings.

| `action` value | Description |
|---|---|
| `"like"` | Participant clicked the like / upvote button |
| `"dislike"` | Participant clicked the dislike / downvote button |
| `"share"` | Participant clicked the share button |
| `"flag"` | Participant flagged the post as potential misinformation |
| `"comment"` | Participant submitted a free-text comment |

<ResponseField name="is_undo" type="boolean">
  When `true`, this reaction reverses a previously recorded action of the same type on the same post. For example, a participant clicking "like" a second time (to unlike) should be sent as `action: "like", is_undo: true`. Your front-end should track toggle state and set this flag accordingly.
</ResponseField>

<ResponseField name="dwell_ms" type="number">
  Time in milliseconds the participant spent viewing the post before this action was recorded. Capture this from first viewport entry to the moment the action fires.
</ResponseField>

<Tip>
In your export data, each row in the reactions sheet contains `is_undo` as a boolean column. When calculating net engagement rates, filter to rows where `is_undo = false`, or compute `SUM(action = 'like' AND is_undo = false) - SUM(action = 'like' AND is_undo = true)` for a net like count per post.
</Tip>

## Study Display Object

The Study Display Object is nested inside the Session bootstrap under the `study` key. It contains all configuration fields your front-end needs to render the participant interface correctly.

<ResponseField name="participant_title" type="string">
  Title displayed at the top of the study interface (e.g. `"Social Media Literacy Study"`).
</ResponseField>

<ResponseField name="consent_text" type="string">
  Full consent form text to show before the participant begins. May contain Markdown.
</ResponseField>

<ResponseField name="instruction_text" type="string">
  Task instructions displayed to participants before the feed. May contain Markdown.
</ResponseField>

<ResponseField name="debrief_text" type="string">
  Debrief text returned again on session completion. May contain Markdown. Also returned by `POST /api/session/complete`.
</ResponseField>

<ResponseField name="show_reactions" type="boolean">
  Master switch for whether any reaction buttons (like, dislike, share, flag, comment) are shown at all.
</ResponseField>

<ResponseField name="enable_comments" type="boolean">
  Whether the comment input is enabled globally for this study. Individual post-level `show_comment` flags can further restrict this per post.
</ResponseField>

<ResponseField name="show_metrics" type="boolean">
  Whether numeric engagement counts (likes, shares, etc.) are shown alongside the reaction buttons.
</ResponseField>

<ResponseField name="layout_type" type="string">
  Controls how posts are presented. Common values include `"feed"` (scrolling feed) and `"single"` (one post at a time with explicit navigation).
</ResponseField>

<ResponseField name="demographics_position" type="string">
  When in the session flow demographics are collected. One of `"before"` (before the feed), `"after"` (after the feed), or `"none"` (demographics disabled).
</ResponseField>

<ResponseField name="clarity_enabled" type="boolean">
  When `true`, the study includes a "clarity" labelling overlay that reveals manipulation techniques to the participant after each post (inoculation design).
</ResponseField>

<ResponseField name="eyetracking_enabled" type="boolean">
  When `true`, the study interface activates eye-tracking data collection (requires participant browser permission and compatible hardware).
</ResponseField>

<ResponseField name="contact_email" type="string">
  Research team contact email displayed on the debrief and consent screens.
</ResponseField>

<ResponseField name="institution" type="string">
  Institution name displayed on the consent screen and debrief.
</ResponseField>
