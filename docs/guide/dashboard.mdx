Each study has its own dashboard — a configurable widget grid that updates as participants complete sessions. The first time you open a study's dashboard, a default layout is generated automatically based on your study design. You can reshape this layout at any time without affecting the underlying data.

The dashboard shows **production sessions only**. Preview sessions — those you ran yourself while testing the study — are excluded from every chart and table on this page.

<Tip>
  The dashboard is designed for **monitoring recruitment**, not for testing hypotheses. Use it to track completion rates, spot dropout patterns, and check that conditions are filling evenly. For statistical inference, use the [Analyses tab](./analyses) or export your data to R, SPSS, JASP, or Python.
</Tip>

## Widget types

The dashboard supports the following widget types. Each widget can be scoped to a specific variable or pair of variables from your study data.

<AccordionGroup>
  <Accordion title="KPIs">
    Single-number summaries: total completed sessions, completion rate, average belief score on false posts, and other configurable metrics.
  </Accordion>
  <Accordion title="Bar charts">
    Counts or means broken down by a categorical variable such as condition, gender, or education level.
  </Accordion>
  <Accordion title="Histograms">
    Distribution of a continuous variable (e.g., dwell time, belief ratings) across all sessions or within a condition.
  </Accordion>
  <Accordion title="Boxplots">
    Median and spread of a continuous variable by group — useful for comparing belief scores across conditions at a glance.
  </Accordion>
  <Accordion title="Crosstabs">
    Frequency table showing the joint distribution of two categorical variables.
  </Accordion>
  <Accordion title="Scatter plots">
    Relationship between two continuous variables, with optional colour-coding by condition.
  </Accordion>
  <Accordion title="Correlation heatmaps">
    Grid of Pearson correlations across a set of numeric variables. Useful for a quick overview of multivariate relationships.
  </Accordion>
  <Accordion title="Time series">
    Session completions over time — by day, week, or custom interval. Helps you spot when a recruitment push drove a surge in responses.
  </Accordion>
  <Accordion title="Text responses">
    Sample of open-text answers to a specific question, displayed as a scrollable list.
  </Accordion>
</AccordionGroup>

## Editing the layout

<Steps>
  <Step title="Enter edit mode">
    Click the **Edit** button (pencil icon) in the dashboard toolbar to enter edit mode.
  </Step>
  <Step title="Add a widget">
    Click **+ Widget**, choose a widget type, select the variable(s) to display, and give the widget an optional title. Click **Save** to place it on the grid.
  </Step>
  <Step title="Rearrange widgets">
    In edit mode, drag any widget to a new position. Widgets snap to the grid and resize automatically to fill available space.
  </Step>
  <Step title="Remove a widget">
    Click the **×** icon on a widget to remove it from the layout. This does not delete any data.
  </Step>
  <Step title="Exit edit mode">
    Click **Done** to save the new layout and return to the live view.
  </Step>
</Steps>

## Filtering the data

### Date-range filter

Use the date-range control in the dashboard toolbar to focus all widgets on a specific time window:

- **7d** — last 7 days
- **30d** — last 30 days
- **All time** — full dataset (default)
- **Custom range** — pick a start and end date from the calendar picker

### Cross-filters

Clicking a bar, segment, or data point in any chart automatically filters all other widgets to show only sessions that match that category. For example, clicking the "Correction" bar in a condition chart narrows every other widget to participants in the Correction condition. Click the same element again to clear the filter.

## Auto-refresh

Use the **Auto-refresh** dropdown to keep the dashboard current during active recruitment periods:

| Setting | Behaviour |
|---|---|
| Off | Data loads once on page open; refresh manually with ⟳ |
| 30 s | Dashboard polls for new sessions every 30 seconds |
| 1 min | Polls every minute |
| 5 min | Polls every 5 minutes |

## Sharing the dashboard

Clicking **Share** generates a **public read-only link** to the dashboard. Anyone with this link can view the charts and filters without being able to edit anything or access the admin panel.

<Note>
  The **Share** button produces a link to the **dashboard**, not to the study itself. Do not use it to recruit participants. The participant link is `/study/<slug>`, which you can find on the study's Settings page.
</Note>

## Exporting a snapshot

You can export the current state of the dashboard — with whatever filters and layout you have applied — as a static image or document:

<CardGroup cols={2}>
  <Card title="PNG snapshot" icon="image">
    Downloads a full-page screenshot of the dashboard grid at the current scroll position. Useful for slide decks and interim reports.
  </Card>
  <Card title="PDF snapshot" icon="file-pdf">
    Downloads a paginated PDF of all widgets. Widgets are reflowed to fit A4 / letter pages automatically.
  </Card>
</CardGroup>

## Recent sessions table

At the bottom of the dashboard, a table lists the **last 20 completed sessions**. Each row shows:

| Column | Description |
|---|---|
| Session ID | Anonymised participant identifier |
| Condition | Full condition label (e.g., `ratio_correction`) |
| Age | Participant age band |
| Gender | Participant gender |
| Avg. belief (false posts) | Mean credibility rating given to false posts in this session |
| Date | Session completion timestamp |

Preview sessions are excluded from this table. If a session was accidentally recorded as production data, you can manage it from the [Export tab](./export).
