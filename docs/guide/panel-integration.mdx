When you field a Misinfo Research study through an external recruiting panel — such as Prolific, Respondi, or a similar agency — you link each participant's session back to the panel's own respondent record. Misinfo Research handles this with two mechanisms: **respondent ID capture** on the way in, and **redirect URLs** on the way out. Both are configured in the study's **Settings** tab (or equivalently in the **Configurator** for builder studies).

<Note>
  Redirect URLs fire only while the study is **active**. Activate the study before distributing the panel link, and deactivate it once data collection is complete so late arrivals are not redirected incorrectly.
</Note>

## Capturing the External Respondent ID

Recruiting panels append a unique respondent identifier to the study URL when sending a participant. Misinfo Research reads this identifier from the URL query string and stores it with the session record.

<ParamField path="respondent_id_param" name="URL parameter name" type="string" default="res_id">
  The name of the query-string parameter that carries the respondent identifier. The default is `res_id`, so a link like `https://your-instance.com/study/your-slug?res_id=ABC123` captures `ABC123`. Change this value if your panel uses a different parameter name (for example, Prolific uses `PROLIFIC_PID`).
</ParamField>

The captured value is stored as **`external_id`** on the session record and appears in the **`external_id` column** of your data export. Use this column to match Misinfo Research responses back to the panel's own completion records.

<Tip>
  Include `{ext_id}` in your completion redirect URL (see below) so the panel receives the respondent's own ID back in the redirect. This is the standard way panels verify that a specific participant finished the study; without it, the panel cannot automatically mark the submission as complete.
</Tip>

## Completion Redirect

After a participant finishes the study, Misinfo Research can redirect them to a panel-supplied completion URL — for example, a Prolific completion link or a Respondi thank-you page.

<ParamField path="completion_redirect_url" name="Completion redirect URL" type="string">
  The URL to send participants to after they complete the study. Supports the placeholders `{ext_id}` (the captured external respondent ID) and `{session_id}` (the Misinfo Research internal session UUID). Example: `https://app.prolific.com/submissions/complete?cc=XXXX&respondent={ext_id}`
</ParamField>

<ParamField path="completion_redirect_delay" name="Redirect delay (seconds)" type="number" default="4">
  How many seconds to wait on the debrief screen before the redirect fires. Set to `0` for an immediate redirect. The default of 4 seconds gives participants a moment to read the debrief before being sent away. The maximum is 600 seconds.
</ParamField>

<ParamField path="completion_redirect_notice" name="Show notice before redirect" type="boolean">
  When enabled, a notice box on the debrief screen informs the participant that they will be redirected. Recommended with a non-zero delay so participants understand why the page changes.
</ParamField>

## Decline Redirect

If a participant reads the consent screen and chooses **not** to participate, you can redirect them back to the panel immediately rather than leaving them on the study's closed screen.

<ParamField path="decline_redirect_url" name="Decline redirect URL" type="string">
  The URL to send participants to when they decline consent. Supports the same placeholders as the completion redirect: `{ext_id}` and `{session_id}`. Many panels require a separate "declined" or "screened-out" link distinct from the completion link — check your panel's documentation.
</ParamField>

<ParamField path="decline_redirect_delay" name="Redirect delay (seconds)" type="number" default="4">
  Seconds to wait before redirecting after a consent decline. The default is 4 seconds. Set to `0` for an immediate redirect. Enabling **immediate redirect** overrides this delay. The maximum is 600 seconds.
</ParamField>

<ParamField path="decline_redirect_immediate" name="Immediate redirect" type="boolean">
  When enabled, participants who decline consent are redirected without seeing any intermediate screen on the Misinfo Research side. Use this when your panel requires seamless flow back to its own "not eligible" or "opted out" page.
</ParamField>

<ParamField path="decline_redirect_notice" name="Show notice before redirect" type="boolean">
  When enabled (and immediate redirect is off), a notice box on the local decline screen appears before the redirect fires. Not applicable when immediate redirect is enabled.
</ParamField>

## Configuration Checklist

<Steps>
  <Step title="Configure the respondent ID parameter">
    In **Settings**, set the **URL parameter name** to match what your panel appends to links. For Prolific this is typically `PROLIFIC_PID`; for Respondi it may differ — check your panel's survey link documentation.
  </Step>
  <Step title="Build your panel study link">
    Share the study URL with the parameter appended. For example: `https://your-instance.com/study/your-slug?PROLIFIC_PID={{%PROLIFIC_PID%}}`. The panel substitutes its own placeholder syntax before sending participants.
  </Step>
  <Step title="Set the completion redirect URL">
    Paste the completion URL provided by your panel into **Completion redirect URL**. Insert `{ext_id}` where the panel expects the respondent identifier. Set a delay and notice if desired.
  </Step>
  <Step title="Set the decline redirect URL">
    Paste the panel's declined/screened-out URL into **Decline redirect URL**. Enable **Immediate redirect** if you do not want participants to see any intermediate screen.
  </Step>
  <Step title="Activate the study">
    Use **Activate** in the action row (see [Settings](/guide/settings)) before distributing the panel link. Redirects do not fire for inactive studies.
  </Step>
  <Step title="Verify with a test session">
    Complete a test run using a dummy respondent ID in the URL. Confirm that the `external_id` column in the export contains the expected value and that the redirect fires correctly at the end of the session.
  </Step>
</Steps>

## Export Reference

| Column | Description |
|---|---|
| `external_id` | The value captured from the configured URL parameter (default `res_id`) for each session. Empty if no parameter was present in the URL. |
| `session_id` | The Misinfo Research internal UUID for the session. Available as the `{session_id}` placeholder in redirect URLs. |

<Tip>
  If you run the same study through multiple panels simultaneously, use a separate slug (duplicated study) per panel so you can track completion rates by source without relying solely on the `external_id` values.
</Tip>
