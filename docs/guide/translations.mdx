Misinfo Research has two independent language layers: the **admin panel UI language**, which controls what you and your team see while building studies, and the **study content language**, which controls what participants see when they take part. Changing one has no effect on the other.

## Admin Panel UI Language

The admin interface is available in **Polish** (the default) and **English**. Switch at any time using the language toggle in the **top-right corner of the header**, or on the **login screen** before signing in. The preference is stored in your browser and applies to your session only — other users on the same instance choose their own language independently.

This setting affects only the labels, buttons, menu items, and messages you see in the admin panel. It does not change anything participants experience.

## Study Content Language

Each study has its own content language, set in **Settings → Language** (see [Settings](/guide/settings)). This controls which translation overlay is shown to participants — consent text, instructions, post headlines and body text, demographic questions, post questions, transition texts, debrief, and action button labels all follow this setting.

Available content languages:

| Code | Language |
|------|----------|
| PL | Polish |
| EN | English |
| CS | Czech |
| SK | Slovak |

Polish is the source language for all study content — your primary text entries are stored in the original Polish columns. When you set a study to a non-Polish language, the platform uses a **translations overlay** (`translations_json` on the study record) to serve the translated strings to participants.

## Translation Workflow

When your study is set to a non-Polish language, every translatable field in the builder and Configurator gains an additional input where you enter the target-language version. You have two options for filling in those fields.

<Steps>
  <Step title="Enter translations manually">
    Type the translated text directly into each target-language field alongside the Polish source. This works for any language and requires no external services — the platform is fully functional without any AI integration.
  </Step>
  <Step title="Use automatic translation">
    Click the **✨ Translate automatically** button at the top of translatable sections. The platform sends the Polish source text to the Anthropic Claude API and populates the target-language fields for you. Review the results and adjust as needed before saving.
  </Step>
</Steps>

<Note>
  Automatic translation requires an `ANTHROPIC_API_KEY` configured on your Misinfo Research instance. Without an API key, the **✨ Translate automatically** button does not appear, and the platform remains fully functional for manual translation. Contact your system administrator to enable AI-assisted translation.
</Note>

### What AI Translation Covers

Automatic translation handles the following content areas in one operation:

<AccordionGroup>
  <Accordion title="Study framework">
    - Consent screen text
    - Instructions shown before the feed
    - Debriefing text shown after completion
    - Transition texts between study parts
    - Part labels
  </Accordion>
  <Accordion title="Posts">
    - Post headlines
    - Post body content
    - Action button labels (e.g. "Share", "Like", custom labels)
    - Post-level questions and their answer options
  </Accordion>
  <Accordion title="Demographic questions">
    - Question labels
    - Answer option labels
  </Accordion>
</AccordionGroup>

Fields left empty by the automatic pass — for example, custom content you add after running the translation — fall back to their Polish source values until you provide a translation manually.

<Tip>
  Run automatic translation early in your workflow, then make manual edits to refine phrasing. Correcting a machine-translated draft is easier than translating from scratch, especially for longer consent and debrief texts.
</Tip>

## Admin Panel Locale Overrides

If the default Polish or English labels in the admin interface do not match your institution's terminology, administrators can customise the UI strings directly. Open the **Tłumaczenia interfejsu** (Interface Translations) modal, accessible from the admin panel settings area.

<Steps>
  <Step title="Open the modal">
    Navigate to the instance-level settings and select **Tłumaczenia interfejsu**. You will see a list of all translatable UI keys for both the Polish and English admin interfaces.
  </Step>
  <Step title="Edit the strings">
    Click any string to edit it. Your override replaces the default value for **all users on this instance** — it is not a per-user setting.
  </Step>
  <Step title="Save changes">
    Overrides are stored in the database and take effect immediately. They persist across platform updates, except when a new release introduces a key that does not yet have an override.
  </Step>
</Steps>

<Note>
  Locale overrides affect the admin panel UI only. They do not modify participant-facing content. Study content translations are managed per study in the builder or Configurator.
</Note>
