The **Analyses** tab gives you access to a set of standard statistical procedures that run directly against your study's production data. You can compute descriptive summaries, compare groups, test associations, and check scale reliability without exporting to an external tool first. This makes it easy to get a quick read on your data while recruitment is still ongoing or immediately after it closes.

<Warning>
  **Built-in analyses are for quick exploration only.** The results shown on this tab have not been independently audited and should not appear in a manuscript without verification. Before reporting any finding for publication, **re-run your analyses in R, SPSS, JASP, or Python** where you have full control over assumptions, effect-size calculations, and reporting standards. Every result on this tab is accompanied by a reminder of this limitation.
</Warning>

## Available analyses

### Descriptive statistics

For any numeric variable in your dataset, the platform computes the mean, standard deviation, sample size (*n*), minimum, and maximum — broken down by group if you select a grouping variable. Use this as your first step before running any inferential test.

### Welch's t-test

Compares the means of a continuous outcome variable between **two groups** (e.g., two conditions, or male vs. female participants). Welch's correction is applied by default so that the test remains valid when the two groups have unequal variances, without requiring you to first run a test of homogeneity of variance.

### One-way ANOVA with Tukey HSD post-hoc

Tests whether the means of a continuous outcome variable differ significantly across **three or more groups**. When the omnibus F-test is significant, **Tukey's Honest Significant Difference** post-hoc comparisons are computed automatically so you can see exactly which pairs of groups differ from each other while controlling the family-wise error rate.

### Pearson and Spearman correlation

Quantifies the linear (Pearson) or rank-based monotonic (Spearman) relationship between two continuous or ordinal variables. Both options return the correlation coefficient *r* (or *ρ*), sample size, and two-tailed *p*-value. Choose Spearman when your variables are ordinal or when a histogram suggests the distribution is heavily skewed.

### Correlation matrix

Extends the pairwise correlation to a full grid of correlations across a set of numeric variables you select. Cells are colour-coded by magnitude and sign, making it easy to spot clusters of related variables at a glance.

### Chi-square test with Cramér's V

Tests whether two categorical variables are independent of each other (e.g., condition and whether participants shared a post). The result includes the χ² statistic, degrees of freedom, *p*-value, and **Cramér's V** as a standardised effect-size measure that is not inflated by large samples the way the raw χ² value can be.

### Linear regression

Fits an ordinary least-squares regression model predicting a continuous outcome from one or more predictor variables. The output table reports unstandardised coefficients, standard errors, *t*-statistics, *p*-values, and overall model *R²*. You can add categorical predictors by selecting a reference category for dummy coding.

### Cronbach's alpha

Estimates the internal consistency reliability of a set of items you believe measure the same latent construct (e.g., a block of credibility-rating questions). The output includes the overall alpha coefficient and the "alpha if item deleted" value for each item, so you can identify items that weaken the scale.

## Running an analysis

<Steps>
  <Step title="Choose a test">
    Select the statistical procedure you want from the left panel of the Analyses tab.
  </Step>
  <Step title="Configure the inputs">
    Choose your outcome variable, grouping variable or predictors, and any relevant options (e.g., Pearson vs. Spearman). Available variables are drawn from the columns in your `Dane_surowe` export sheet, including session metadata, demographics, per-post reactions, ratings, and question answers.
  </Step>
  <Step title="Run the analysis">
    Click **Run**. Results appear in the main panel within a few seconds for typical sample sizes.
  </Step>
  <Step title="Review the exploration warning">
    Each results panel displays a reminder that these outputs are for exploratory use only. Note this before sharing screenshots or summary tables with co-authors.
  </Step>
</Steps>

## Saving analyses for later

You can save any configured analysis — its test type, variable selections, and options — so that you can re-run it as new sessions come in or revisit it after a period of time.

<Steps>
  <Step title="Click Save analysis">
    After running an analysis, click the **Save analysis** button in the results panel.
  </Step>
  <Step title="Give it a name">
    Enter a short descriptive label (e.g., "Condition × Belief — Welch t-test") and click **Save**.
  </Step>
  <Step title="Re-run from the saved list">
    Saved analyses appear in a list on the left panel. Click any saved item and then **Run** to execute it against the current dataset. The configuration is restored exactly as you left it.
  </Step>
</Steps>

<Tip>
  Saving analyses is especially useful during ongoing data collection. You can re-run the same set of tests each week to monitor how effect estimates stabilise as your sample grows — while remembering that sequential peeking inflates Type I error rates, which is one more reason to treat these outputs as exploratory.
</Tip>

## Recommended external tools

Once your data collection is complete, carry your analysis into one of the following environments for publication-quality results:

<CardGroup cols={2}>
  <Card title="R / RStudio" icon="r-project">
    Full-featured statistical computing with packages such as `lme4`, `effectsize`, `psych`, and `ggplot2`. Ideal for complex mixed models and reproducible scripting.
  </Card>
  <Card title="JASP" icon="chart-bar">
    Point-and-click interface that mirrors the Analyses tab experience while producing APA-formatted output tables and Bayesian alternatives for every frequentist test.
  </Card>
  <Card title="SPSS" icon="table">
    Widely used in social-science departments. Supports all tests available on the Analyses tab and generates syntax files for reproducibility.
  </Card>
  <Card title="Python (pingouin / scipy)" icon="python">
    The `pingouin` library replicates every test on this tab and integrates with `pandas` DataFrames, making it easy to load your exported CSV directly.
  </Card>
</CardGroup>
