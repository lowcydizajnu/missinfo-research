The **Export** tab is your starting point for getting data out of Misinfo Research and into your analysis environment. Before you download anything, the tab shows three session counts at the top of the page: **completed sessions**, **total sessions** (including drop-outs), and **preview sessions**. These numbers help you understand what is in the export before you commit to a download format.

Only **completed sessions** are included in exports. Participants who started the study but did not reach the final screen are counted in the dashboard and reflected in the "total sessions" figure on this tab, but they are never written to an export file.

## Download formats

<CardGroup cols={2}>
  <Card title="Excel (.xlsx)" icon="file-excel">
    A multi-sheet workbook containing raw reactions, credibility ratings, session summaries, a design pivot table, and supplementary sheets for questions, demographics, and optional modules. This is the recommended format for most research workflows.
  </Card>
  <Card title="CSV" icon="file-csv">
    A single file containing the raw-data sheet (`Dane_surowe`) only. Use this when you are importing directly into R, Python, or another tool that reads delimited text.
  </Card>
</CardGroup>

## Excel workbook structure

The Excel export is organised into named sheets. The table below describes each sheet and what it contains.

| Sheet name | Contents |
|---|---|
| `Dane_surowe` | One row per participant reaction. Columns: `session_id`, `post_id`, `post_order`, `action` (like / dislike / share / flag / comment), `dwell_ms`, `likes_shown`, `shares_shown`, `dislikes_shown`, `flags_shown`, `style_condition`, `metric_condition`, `full_condition`, `is_true`, `is_undo`. |
| `Oceny_wiarygodnosci` | One row per credibility rating. In classic studies this is a `belief_1_7` scale entry per post per session. Builder studies use post-level questions instead of this sheet. |
| `Podsumowanie_sesji` | One row per completed session with aggregate metrics: `session_id`, `started_at`, `completed_at`, condition columns, all demographic fields, average reaction rates, average belief on true posts, and average belief on false posts. |
| `Design_2x2` | Pivot table showing cell sizes and a demographic breakdown for each condition cell. Useful for verifying that randomisation distributed participants evenly. |
| Post questions | One sheet per post-level question block, one row per answer. Present when post questions are configured in the study. |
| Demographic questions | Custom demographic items beyond the standard coding-key variables. |
| Eye tracking | Gaze fixation and saccade data per post per session. Present only when eye-tracking is enabled for the study. |
| Coding key | Human-readable legend for all numeric demographic codes (see below). |

## Demographic coding key

Numeric codes are used throughout the export to represent demographic responses. The **Coding key** sheet documents each code. A summary of the standard variables is below:

<AccordionGroup>
  <Accordion title="Age">
    Age is recorded as a band code rather than an exact year. The coding key sheet lists the age range corresponding to each numeric value.
  </Accordion>
  <Accordion title="Gender">
    Gender is stored as a small integer. The coding key maps each value to its label (e.g., woman, man, non-binary, prefer not to say).
  </Accordion>
  <Accordion title="Education">
    Education level is an ordered integer scale from primary / secondary through postgraduate. The coding key documents each step.
  </Accordion>
  <Accordion title="Residence">
    Residence type is coded as an integer representing settlement size categories (e.g., rural, small town, large city). The coding key sheet lists all categories.
  </Accordion>
</AccordionGroup>

## Column builder

Before downloading, you can open the **column builder** to customise exactly which columns appear in your export and how they are labelled.

<Steps>
  <Step title="Open the column builder">
    Click **Preview and customize columns** on the Export tab. An inline panel opens showing all available columns for the selected download format.
  </Step>
  <Step title="Hide columns you do not need">
    Toggle the visibility switch next to any column to hide it from the export. Hidden columns remain in the database — they simply do not appear in the downloaded file.
  </Step>
  <Step title="Rename columns">
    Click a column's name to edit it inline. Rename columns to match the variable names expected by your analysis scripts.
  </Step>
  <Step title="Reorder columns">
    Drag rows in the builder to reorder columns in the output file.
  </Step>
  <Step title="Save as a profile">
    Click **Save profile** and give the arrangement a name. Saved profiles appear in a dropdown at the top of the builder so you can apply the same layout in future download sessions without reconfiguring from scratch.
  </Step>
</Steps>

<Warning>
  **Hiding a column in the column builder never deletes data.** The underlying values are always preserved in the database. If you need a column you previously hid, simply reopen the builder and toggle it back on, or switch to a different profile.
</Warning>

## Preview sessions

Preview sessions — those generated by clicking the **Preview** button while testing your study — are excluded from all exports by default.

<AccordionGroup>
  <Accordion title="Include preview sessions in the export">
    Check the **Include completed preview sessions** checkbox on the Export tab before downloading. Preview sessions are marked with `is_preview = 1` in the `Dane_surowe` sheet so you can filter them out in your analysis tool if needed.
  </Accordion>
  <Accordion title="Delete all preview sessions">
    Click **Delete preview sessions** to permanently remove all preview sessions and their associated reactions, ratings, and question answers. This action does **not** affect any production session. Use it to clean up your export counts before sharing data with collaborators.
  </Accordion>
</AccordionGroup>

<Note>
  The session counts at the top of the Export tab reflect the state of the database at the moment you load the page. After deleting preview sessions, refresh the page to see updated counts.
</Note>
