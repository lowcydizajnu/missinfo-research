The **Export** tab is your starting point for getting data out of Misinfo Research and into your analysis environment. Before you download anything, the tab shows three session counts at the top of the page: **completed sessions**, **total sessions** (including drop-outs), and **preview sessions**. These numbers help you understand what is in the export before you commit to a download format.

Only **completed sessions** are included in exports. Participants who started the study but did not reach the final screen are counted in the dashboard and reflected in the "total sessions" figure on this tab, but they are never written to an export file.

## Download formats

<CardGroup cols={2}>
  <Card title="Excel (.xlsx)" icon="file-excel">
    A workbook with two sheets: `Dane_surowe` (one wide row per session, with every post's reactions, ratings, and question answers pivoted into columns) and `Klucz_kodowania` (the demographic and condition codebook). This is the recommended format for most research workflows.
  </Card>
  <Card title="CSV" icon="file-csv">
    A single file containing the `Dane_surowe` sheet only, in comma-separated format. Use this when you are importing directly into R, Python, or another tool that reads delimited text.
  </Card>
</CardGroup>

## Excel workbook structure

The Excel export contains two sheets.

| Sheet name | Contents |
|---|---|
| `Dane_surowe` | One row per completed session. Each post appears as a group of columns with a numeric prefix (e.g. `post_1_reaction`, `post_1_dwell_ms`, `post_2_belief_1_7`). The sheet also includes session metadata (`session_id`, `full_condition`, `style_condition`, `metric_condition`, `started_at`, `completed_at`, `duration_min`), all demographic fields with their numeric codes, any post-level question answers, and a conditional-logic audit trail (`logic_applied`, `logic_skipped_parts`, `logic_end_rule`). |
| `Klucz_kodowania` | Human-readable legend that maps every numeric demographic code to its label (e.g. gender `1` = woman, age `2` = 26–35). Also lists all condition labels and post-question IDs so you can decode the `Dane_surowe` columns without referring back to the platform. |

## Demographic coding key

Numeric codes are used throughout the export to represent demographic responses. The **`Klucz_kodowania`** sheet documents each code. A summary of the standard variables is below:

<AccordionGroup>
  <Accordion title="Age">
    Age is recorded as a band code rather than an exact year. The `Klucz_kodowania` sheet lists the age range corresponding to each numeric value.
  </Accordion>
  <Accordion title="Gender">
    Gender is stored as a small integer. The `Klucz_kodowania` sheet maps each value to its label (e.g., woman, man, non-binary, prefer not to say).
  </Accordion>
  <Accordion title="Education">
    Education level is an ordered integer scale from primary / secondary through postgraduate. The `Klucz_kodowania` sheet documents each step.
  </Accordion>
  <Accordion title="Residence">
    Residence type is coded as an integer representing settlement size categories (e.g., rural, small town, large city). The `Klucz_kodowania` sheet lists all categories.
  </Accordion>
</AccordionGroup>

## Column builder

Before downloading, you can open the **column builder** to customise exactly which columns appear in your export and how they are labelled.

<Steps>
  <Step title="Open the column builder">
    Click **Preview and customize columns** on the Export tab. An inline panel opens showing all available columns for the selected download format.
  </Step>
  <Step title="Hide columns you do not need">
    Toggle the visibility switch next to any column to hide it from the export. Hidden columns remain in the database — they simply do not appear in the downloaded file.
  </Step>
  <Step title="Rename columns">
    Click a column's name to edit it inline. Rename columns to match the variable names expected by your analysis scripts.
  </Step>
  <Step title="Reorder columns">
    Drag rows in the builder to reorder columns in the output file.
  </Step>
  <Step title="Save as a profile">
    Click **Save profile** and give the arrangement a name. Saved profiles appear in a dropdown at the top of the builder so you can apply the same layout in future download sessions without reconfiguring from scratch.
  </Step>
</Steps>

<Info>
  **Hiding a column in the column builder never deletes data.** The underlying values are preserved in the database. To bring back a hidden column, reopen the builder and toggle it back on, or switch to a different profile.
</Info>

## Preview sessions

Preview sessions — those generated by clicking the **Preview** button while testing your study — are excluded from all exports by default.

<AccordionGroup>
  <Accordion title="Include preview sessions in the export">
    Check the **Include completed preview sessions** checkbox on the Export tab before downloading. Preview sessions are marked with `is_preview = 1` in the `Dane_surowe` sheet so you can filter them out in your analysis tool if needed.
  </Accordion>
  <Accordion title="Delete all preview sessions">
    Click **Delete preview sessions** to permanently remove all preview sessions and their associated reactions, ratings, and question answers. This action does **not** affect any production session. Use it to clean up your export counts before sharing data with collaborators.
  </Accordion>
</AccordionGroup>

<Note>
  The session counts at the top of the Export tab reflect the state of the database at the moment you load the page. After deleting preview sessions, refresh the page to see updated counts.
</Note>
