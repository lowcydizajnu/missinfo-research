Railway is a managed cloud platform that lets you deploy Misinfo Research without configuring servers or container runtimes yourself. It auto-detects the Node.js application via `package.json`, provisions a public URL, and supports persistent volumes for your database and uploads.

## One-click deploy

The fastest way to get started is the Railway template link. Click it to open a pre-configured deployment in your Railway dashboard:

[**Deploy Misinfo Research on Railway →**](https://railway.com/new/template?template=https://github.com/lowcydizajnu/missinfo-research)

You need a Railway account (free tier available). If you are not already signed in, Railway prompts you to log in or sign up before continuing.

## Configuration

<Steps>
  <Step title="Open the template deploy page">
    Follow the one-click link above. Railway shows a summary of the services the template creates, along with a form to set environment variables before the first deploy.
  </Step>

  <Step title="Set required environment variables">
    In the **Variables** section, add the following before clicking **Deploy**:

    | Variable | Value |
    |---|---|
    | `JWT_SECRET` | A long random string — generate one with `node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"` on any machine with Node.js |
    | `ADMIN_PASSWORD` | The password for the `admin` account created on first boot |
    | `NODE_ENV` | `production` |

    <Info>
      Set `JWT_SECRET` before the first deploy. Without it, the app exits on startup. You can add it later in the Railway dashboard and redeploy, but setting it upfront avoids the extra step.
    </Info>
  </Step>

  <Step title="Deploy the application">
    Click **Deploy**. Railway clones the repository, installs dependencies, and starts the Node.js process. Auto-detection picks up `package.json`, so no extra build configuration is required.

    You can watch the build and deploy logs in real time from the Railway dashboard.
  </Step>

  <Step title="Add a persistent volume for the database">
    Railway's filesystem is ephemeral: files written outside a mounted volume are lost when the service restarts. Attach a volume to persist the SQLite database.

    In your service settings on Railway:

    1. Go to **Settings → Volumes** (or **Storage** depending on your dashboard version).
    2. Add a new volume and set the **mount path** to `/app/data`.
    3. Save and redeploy the service.

    This volume stores `research.db` and survives all future deploys and restarts.

    <Info>
      To store uploaded images, add a second volume mounted at `/app/uploads`. Without it, uploaded files are lost on each redeploy.
    </Info>
  </Step>

  <Step title="Access the admin panel">
    Once the deploy succeeds, Railway assigns your service a public URL of the form `https://yourapp.railway.app`. Navigate to:

    ```
    https://yourapp.railway.app/admin
    ```

    Log in with:
    - **Username:** `admin`
    - **Password:** the value you set for `ADMIN_PASSWORD`

    <Tip>
      Find or customise your public URL under **Settings → Networking → Public Domain** in the Railway dashboard.
    </Tip>
  </Step>
</Steps>

## Updating environment variables after deploy

To change a variable after the initial deploy:

1. Open your service in the Railway dashboard.
2. Go to the **Variables** tab.
3. Edit or add the variable and click **Save**.
4. Railway automatically triggers a redeploy with the new values.

<Info>
  Changing `ADMIN_PASSWORD` has no effect on an existing `admin` account — the seed runs only when no admin account exists. Use the admin panel to change the password instead.
</Info>
