Installing Misinfo Research directly on Node.js gives you the most control over your environment. This guide walks you through every step, from verifying your prerequisites to logging in to the admin panel for the first time.

## Prerequisites

Before you begin, make sure the following are available on your server or local machine:

- **Node.js 18 or later** — check with `node -v`
- **npm** — bundled with Node.js; check with `npm -v`
- **git** — to clone the repository
- **Writable disk** — the application creates a SQLite database and an uploads directory on first run

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the project from GitHub and move into the new directory:

    ```bash
    git clone https://github.com/lowcydizajnu/missinfo-research.git
    cd missinfo-research
    ```
  </Step>

  <Step title="Install dependencies">
    Install all Node.js dependencies with npm:

    ```bash
    npm install
    ```
  </Step>

  <Step title="Create your environment file">
    Copy the documented template to create your local configuration file:

    ```bash
    cp .env.example .env
    ```

    Open `.env` in a text editor — every available variable is described in the template. You will fill in the required values in the next step.
  </Step>

  <Step title="Set required environment variables">
    Two variables must be present before the server will start.

    **`JWT_SECRET`** — a long, random string used to sign admin session tokens. Generate one now with Node's built-in `crypto` module:

    ```bash
    node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
    ```

    Copy the output and paste it as the value of `JWT_SECRET` in your `.env` file.

    **`ADMIN_PASSWORD`** — the password that will be set on the `admin` account the first time the server boots. Choose a strong password and record it somewhere safe.

    Your `.env` should contain at minimum:

    ```env
    JWT_SECRET=your_generated_secret_here
    ADMIN_PASSWORD=your_strong_password_here
    ```

    <Warning>
      The server will refuse to start if `JWT_SECRET` is missing or empty. Keep this value secret — anyone who knows it can forge admin tokens.
    </Warning>
  </Step>

  <Step title="Configure optional environment variables">
    The following variables are optional. You can leave them unset to accept the defaults, or add them to your `.env` file as needed:

    | Variable | Default | Purpose |
    |---|---|---|
    | `PORT` | `3000` | HTTP port the server listens on |
    | `NODE_ENV` | — | Set to `production` on hosted servers |
    | `DATA_PATH` | `./data` | Directory where the SQLite database is stored |
    | `UPLOADS_PATH` | `./uploads` | Directory where uploaded images are stored |
    | `ANTHROPIC_API_KEY` | — | Enables AI-assisted translation features |

    See [Environment Variables](/hosting/configuration) for full details on each option.
  </Step>

  <Step title="Start the server">
    Choose the appropriate command for your use case:

    <CodeGroup>
      ```bash Production
      npm start
      ```

      ```bash Development (auto-reload)
      npm run dev
      ```
    </CodeGroup>

    On first run, the server automatically:
    - Creates the SQLite database in `./data/` (or `DATA_PATH`)
    - Runs all schema migrations
    - Seeds an admin account with the username `admin` and the password you set in `ADMIN_PASSWORD`

    <Note>
      The admin account is only seeded if no admin account exists yet. If you restart the server later with a different `ADMIN_PASSWORD`, the existing account is not affected.
    </Note>
  </Step>

  <Step title="Log in to the admin panel">
    Open your browser and navigate to:

    ```
    http://localhost:3000/admin
    ```

    Log in with:
    - **Username:** `admin`
    - **Password:** the value you set for `ADMIN_PASSWORD`

    <Tip>
      If you changed the default `PORT`, replace `3000` in the URL with your chosen port number.
    </Tip>
  </Step>
</Steps>
