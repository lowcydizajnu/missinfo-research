Misinfo Research is configured entirely through environment variables. In a Node.js or Docker installation these are set in a `.env` file in the project root. On Railway and other cloud platforms you set them through the platform's dashboard. This page documents every variable the application recognises.

<Warning>
  Never commit your `.env` file to git. It contains secrets such as `JWT_SECRET` and `ADMIN_PASSWORD` that grant full administrative access to your instance. The repository's `.gitignore` already excludes `.env`, but double-check before pushing to a remote.
</Warning>

## Variable reference

| Variable | Required | Default | Description |
|---|---|---|---|
| `JWT_SECRET` | **Yes** | — | Long random string used to sign and verify admin JWT tokens. Sessions expire after 24 hours. The server refuses to start if this is missing or empty. |
| `ADMIN_PASSWORD` | **Yes** | — | Password assigned to the `admin` account when the database is first seeded. Has no effect once an admin account already exists. |
| `PORT` | No | `3000` | TCP port the HTTP server listens on. |
| `NODE_ENV` | No | — | Set to `production` on any publicly accessible or hosted instance to enable production-mode optimisations and error handling. |
| `DATABASE_PATH` | No | `./data/research.db` | Full file path for the SQLite database. The directory must be writable by the Node.js process. |
| `UPLOADS_PATH` | No | `./uploads` | Directory where user-uploaded images are stored. Must be writable by the Node.js process. |
| `ANTHROPIC_API_KEY` | No | — | API key for the Anthropic Claude API. When present, enables AI-assisted translation features across the platform. |

## Required variables in detail

### `JWT_SECRET`

This secret is used to sign every admin session token. If it changes, all existing sessions are immediately invalidated and admins must log in again. Use a value that is long, random, and never reused across instances.

Generate a suitable value with Node.js:

```bash
node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
```

Or with OpenSSL if Node.js is not available:

```bash
openssl rand -hex 48
```

### `ADMIN_PASSWORD`

This password seeds the initial `admin` account on the very first startup. It is only applied if no admin account exists in the database yet — subsequent restarts (even with a changed value) do not overwrite the stored password.

To change the admin password after the first boot, use the admin panel rather than this variable.

## Optional variables in detail

### `DATABASE_PATH`

Override the default database file path when you need the database on a specific volume or mount point. This variable takes the full path to the `.db` file, not just a directory. Example:

```env
DATABASE_PATH=/mnt/persistent/data/research.db
```

In Docker deployments the named volume `misinfo-data` is mounted at `/app/data`, so the database lands at `/app/data/research.db` by default. Only set this variable if you need a non-standard path.

### `UPLOADS_PATH`

Override the default uploads directory. Example:

```env
UPLOADS_PATH=/mnt/persistent/uploads
```

In Docker deployments the named volume `misinfo-uploads` is mounted at `/app/uploads`.

### `ANTHROPIC_API_KEY`

When this key is set, the platform can call the Anthropic Claude API to provide AI-assisted translation of experiment content. If the key is absent, translation features are disabled but the rest of the platform functions normally.

## Example `.env` file

```env
# Required
JWT_SECRET=a1b2c3d4e5f6...your_long_random_string_here
ADMIN_PASSWORD=change_me_before_deploying

# Recommended for hosted instances
NODE_ENV=production

# Optional overrides
PORT=3000
DATABASE_PATH=./data/research.db
UPLOADS_PATH=./uploads

# Optional integrations
ANTHROPIC_API_KEY=sk-ant-...
```

<Tip>
  Run `cp .env.example .env` to start from the documented template included in the repository. All variables are listed there with inline comments explaining each one.
</Tip>
